## рҹҺҜ What Is This?

> Single LLM = one perspective. This project gives you **three agents arguing, countering, and judging** вҖ” like a real debate.

Most AI apps ask one model for one answer.
This system demonstrates **Agentic AI orchestration** вҖ” where multiple LLMs:

```
  рҹҹў Think independently      вҶ’   Each agent has its own role & prompt
  рҹ”„ Share context             вҶ’   Memory flows across agents
  вҡ”пёҸ  Compete & challenge      вҶ’   FOR vs AGAINST dynamics
  вҡ–пёҸ  Get evaluated            вҶ’   JUDGE agent gives a structured verdict
```

**Result:** More structured, critical, and explainable AI reasoning.

## рҹҡҖ Core Features

### рҹ§  Multi-Agent Orchestration
Three specialized agents with distinct roles, each operating on a tailored system prompt and receiving prior agent context before generating output.

| Agent | Role | Behavior |
|-------|------|----------|
| рҹҹў **FOR** | Advocate | Builds the strongest case in favor |
| рҹ”ҙ **AGAINST** | Challenger | Counters with opposing logic |
| вҡ–пёҸ **JUDGE** | Evaluator | Scores both sides, delivers verdict |

### вҡЎ Real-Time Token Streaming
Output is streamed token-by-token using Python `yield` вҖ” giving a live, low-latency debate experience directly in the Gradio UI.

### рҹ”„ Shared Context System
Each agent receives the full output of previous agents before responding вҖ” enabling coherent, multi-step reasoning rather than isolated answers.

### рҹ”’ Fully Local & Private
- Runs entirely on **Ollama (llama3.2)** вҖ” no API keys, no cloud calls
- All processing stays on your machine
- Zero data leaves your environment

### рҹ–ҘпёҸ Interactive Gradio UI
Clean `ChatInterface` layout вҖ” just type a topic and watch the debate unfold.

## вҡҷпёҸ Installation & Setup

### 1пёҸвғЈ Clone the Repository

```bash
git clone https://github.com/TheShivaji/llm-multi-agent-debate.git
cd llm-multi-agent-debate
```

### 2пёҸвғЈ Install Python Dependencies

```bash
pip install -r requirements.txt
```

### 3пёҸвғЈ Pull & Start the Local LLM

```bash
# Pull the model (first time only)
ollama pull llama3.2

# Start the model server
ollama run llama3.2
```

> рҹ’Ў Make sure Ollama is running at `http://localhost:11434` before launching the app.

### 4пёҸвғЈ Launch the App

```bash
python app.py
```

App opens at вҶ’ `http://localhost:7860`
